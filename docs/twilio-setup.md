# Twilio SMS Setup

This guide covers local development setup for the Twilio SMS integration used in S3. Production configuration is deferred to S6.

## 1. Create a Twilio Account

Go to [twilio.com](https://www.twilio.com) and sign up for a free trial account. The trial includes enough credit to test SMS locally.

## 2. Purchase a Canadian Phone Number

In the Twilio Console:

1. Navigate to **Phone Numbers > Manage > Buy a number**
2. Set the country to **Canada**
3. Under **Capabilities**, enable **SMS** and **MMS**
4. Purchase the number

Note the number in E.164 format (e.g. `+16135550100`) — you will need it for the env var.

## 3. Configure the Incoming SMS Webhook

In the Twilio Console:

1. Go to **Phone Numbers > Manage > Active numbers** and click your number
2. Under **Messaging Configuration**, set the webhook for **A message comes in** to:
   ```
   POST https://{ngrok-url}/api/sms
   ```
   Replace `{ngrok-url}` with the public URL generated by ngrok (see step 4).
3. Save the configuration.

## 4. Install and Run ngrok

ngrok exposes your local dev server to the public internet so Twilio can reach the webhook.

```bash
# Install via Homebrew (macOS)
brew install ngrok

# Authenticate once (get token from https://dashboard.ngrok.com/get-started/your-authtoken)
ngrok config add-authtoken <your-token>

# Start a tunnel to the Next.js dev server
ngrok http 3000
```

ngrok will print a forwarding URL like `https://abc123.ngrok-free.app`. Use this as `{ngrok-url}` in the webhook URL above.

Leave the ngrok process running for the duration of your test session. The URL changes each restart, so update the Twilio webhook URL whenever you restart ngrok.

## 5. Set Environment Variables

Copy `.env.example` to `.env.local` if you have not already, then fill in your Twilio credentials:

```bash
cp .env.example .env.local
```

Open `.env.local` and set:

```
TWILIO_ACCOUNT_SID=ACxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
TWILIO_AUTH_TOKEN=your-auth-token
TWILIO_PHONE_NUMBER=+1XXXXXXXXXX
```

Find `TWILIO_ACCOUNT_SID` and `TWILIO_AUTH_TOKEN` on the [Twilio Console homepage](https://console.twilio.com).

## 6. Test the Integration

1. Start the Next.js dev server: `npm run dev`
2. Start ngrok: `ngrok http 3000`
3. Update the Twilio webhook URL with the new ngrok URL (if changed)
4. Send an SMS to your Twilio number from any phone
5. Open the ngrok inspector at `http://localhost:4040` — you should see an incoming `POST /api/sms` request with the message payload

## Production Configuration

Production webhook and phone number configuration is out of scope for S3 and will be handled in S6.
